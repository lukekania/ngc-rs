# Changelog

All notable changes to ngc-rs are documented here. The format follows
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/) and this project
adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Per-tag GitHub Releases are auto-generated by `.github/workflows/release-notes.yml`
from conventional-commit messages — this file captures the higher-level narrative
across milestones.

publish anything to npm.

## [0.10.0] — 2026-05-04

The architecture for the v1.0 drop-in landed (application builder + npm
distribution + cross-compile release pipeline + budget enforcement +
full minification), but a feature-parity audit against `@angular/build:application`
on Angular 21 surfaced several `angular.json` options that ngc-rs doesn't
yet honor. Rather than ship under-specified, the v1.0.0 tag is held until
the parity gap closes; the work landed below ships as v0.10.0 for internal
version tracking.

The original "v1.0.0" entry that lived in this changelog at commit time
has been re-labeled as v0.10.0 to keep semver honest — v1.0.0 was never
published. See the [v1.0 — Angular CLI parity](https://github.com/lukekania/ngc-rs/milestone/13)
milestone for the gap list and progress.

### Added

- **`@ngc-rs/builder:application`** — application builder counterpart to the
  existing `dev-server` builder. Speaks the `@angular-devkit/architect`
  protocol; spawns the `ngc-rs` binary with `--output-json` and surfaces
  diagnostics through `BuilderContext.logger`.
- **`@ngc-rs/cli`** — npm wrapper package distributing the Rust binary via
  five platform-specific `optionalDependencies` (esbuild/biome/swc pattern):
  `@ngc-rs/cli-darwin-arm64`, `-darwin-x64`, `-linux-arm64`, `-linux-x64`,
  `-win32-x64`. No postinstall, no network during install.
- **`angular.json` `budgets` enforcement** — parses the `budgets` array
  (`initial`, `anyComponentStyle`, `anyScript`, `bundle`, `all`, `allScript`)
  and fails the build with a `ng build`-style diagnostic table when a
  `maximumError` threshold is exceeded.
- **Cross-compile release pipeline** — `cargo dist`-driven workflow that
  produces archives + checksums for five targets (linux-{x64,arm64},
  darwin-{x64,arm64}, win-x64) on every `v*` tag, then publishes the npm
  packages with provenance attestations.
- **Public-repo polish** — `SECURITY.md`, `CODE_OF_CONDUCT.md` (Contributor
  Covenant 2.1), GitHub issue templates, PR template, `CHANGELOG.md`.
- **Workflow hardening** — every third-party action SHA-pinned, minimal
  `permissions:` blocks, tag-only secret access, branch + tag protection
  rulesets, "require approval for outside collaborators" gating.

### Changed

- **License** — relicensed from MIT to **MIT OR Apache-2.0** dual (Rust
  ecosystem convention; adds Apache-2.0's explicit patent grant).
- **Production minification** — replaced whitespace-only `oxc_codegen --minify`
  with the full `oxc_minifier` pipeline (constant folding, dead-code
  elimination, identifier mangling). Closes ~3× of the bundle-size gap with
  vanilla `ng build`.
- **`--output-json` shape** — reshaped to match `@angular-devkit/architect`'s
  `BuilderOutput`: `success`, `error`, `errors[]`, `warnings[]`,
  `output_path`, `output_files[{path,size,kind}]`, `modules_bundled`,
  `total_size_bytes`, `duration_ms`. The failure path also emits this shape
  before exiting non-zero so the architect shim can parse a coherent result
  regardless of exit code.
- **Workspace version** bumped 0.9.6 → 1.0.0; all npm packages set to 1.0.0.

### Known limitations

- **Initial bundle size is ~50% larger than `ng build`'s** — vendor chunk
  splitting (carving npm code out of `main.js` into separate cacheable
  chunks) is not implemented. Functionality is correct and 6× faster, but
  more bytes on first load. Tracked as [#131](https://github.com/lukekania/ngc-rs/issues/131)
  against the v1.0.1 milestone.
- **`anyComponentStyle` budgets** are evaluated against the global `styles.css`
  bundle rather than per-component-style bundles (ngc-rs doesn't yet emit
  per-component style bundles separately).
- **Library crates not published to crates.io** — the workspace crates form
  a tightly-coupled toolchain rather than reusable libraries. Rust users
  install via `cargo install --git https://github.com/lukekania/ngc-rs --tag v1.0.0`.
